title: "Quick Start"
description: "Build and run your first Motus agent in under five minutes."
---

This guide walks you through creating a working agent that calls a tool to answer a question. By the end you will have a complete Python script you can run locally.

<Steps>
  <Step title="Create a tool">
    A tool is any async Python function decorated with `@tool`. The docstring becomes the description the LLM uses to decide when and how to call it.

    ```python
    from motus.tools import tool

    @tool
    async def weather(city: str) -> str:
        """Get the current weather for a city."""
        # In a real app, call a weather API here
        return f"It is 22°C and sunny in {city}."
    ```

    Type annotations on parameters are required — Motus uses them to generate the JSON schema the model sees.
  </Step>

  <Step title="Create a model client">
    Motus provides a unified client interface across providers. Pick the one you have an API key for:

    <Tabs>
      <Tab title="OpenAI">
        ```python
        from motus.models import OpenAIChatClient

        client = OpenAIChatClient(api_key="sk-...")
        model = "gpt-4o"
        ```
      </Tab>
      <Tab title="Anthropic">
        ```python
        from motus.models import AnthropicChatClient

        client = AnthropicChatClient(api_key="sk-ant-...")
        model = "claude-sonnet-4-20250514"
        ```
      </Tab>
      <Tab title="OpenRouter">
        ```python
        from motus.models import OpenRouterChatClient

        client = OpenRouterChatClient(api_key="sk-or-...")
        model = "openai/gpt-4o"
        ```
      </Tab>
    </Tabs>

    You can also omit the `api_key` argument — the client falls back to the corresponding environment variable (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, or `OPENROUTER_API_KEY`).
  </Step>

  <Step title="Create an agent">
    `ReActAgent` combines a model client, a model name, and a list of tools into a reasoning loop. Pass an optional `system_prompt` to give the agent a persona or standing instructions.

    ```python
    from motus.agent import ReActAgent

    agent = ReActAgent(
        client,
        model,
        system_prompt="You are a helpful weather assistant.",
        tools=[weather],
    )
    ```
  </Step>

  <Step title="Run the agent">
    Agents are async callables. Pass a user message and `await` the final text response:

    ```python
    import asyncio

    async def main():
        response = await agent("What's the weather like in Tokyo?")
        print(response)

    asyncio.run(main())
    ```

    Expected output:

    ```
    It is 22°C and sunny in Tokyo.
    ```

    The agent received your question, decided to call the `weather` tool with `city="Tokyo"`, incorporated the result, and returned a natural-language answer.
  </Step>

  <Step title="Serve locally (optional)">
    To expose your agent as an HTTP API, run the Motus server:

    ```bash
    motus serve start myapp:agent --port 8000
    ```

    Replace `myapp:agent` with the Python module path and attribute name of your agent instance. The server starts a FastAPI app at `http://localhost:8000`.
  </Step>
</Steps>

## Full example

Here is the complete script from the steps above in a single file:

```python
import asyncio
from motus.agent import ReActAgent
from motus.models import OpenAIChatClient
from motus.tools import tool

@tool
async def weather(city: str) -> str:
    """Get the current weather for a city."""
    return f"It is 22°C and sunny in {city}."

client = OpenAIChatClient(api_key="sk-...")
agent = ReActAgent(
    client,
    "gpt-4o",
    system_prompt="You are a helpful weather assistant.",
    tools=[weather],
)

async def main():
    print(await agent("What's the weather like in Tokyo?"))

asyncio.run(main())
```
